# Hands-on nata.house

O Hands-on nata.house é um formato de apresentações focadas no teor prático. O seu objetivo é alinhar com todos os colaboradores da empresa os conhecimentos técnicos e as boas práticas de desenvolvimento de software, além de promover a transferência de conhecimentos entre os diferentes membros do time.

## Objetivo

- Alinhar tecnicamente e culturalmente a equipe
- Promover transferência de conhecimentos diferentes entre membros
- Fortalecer os projetos realizados na empresa, trazendo códigos mais robustos e arquiteturas mais consistentes
- Reduzir tempo gasto com alinhamento de código no dia-a-dia

## Principios

- Não se trata de uma palestra
- A apresentação é de caráter prático (muitos exemplos de código)
- Somente sugerir temas relevantes para toda a equipe


## Regras

- Ocorre pelo menos 1x por mês, com o máximo de duas vezes por mês. Sempre com espaços de duas semanas ou mais entre as apresentações.
- O tema deve ser validado primeiramente no grupo do telegram e depois sugerido para o administrativo. O administrativo é responsável por marcar o dia e reservar o auditório.
- As apresentações devem ter 40 minutos de duração máxima. Com espaço para dúvidas.
- Esteja treinado para sua apresentação. Caso sejá código, é ideal você ter suas colas já implementadas.
- Utilize o padrão de slides definido para este projeto. Os slides deverão ser disponibilizados para toda a empresa após.
- Divida a apresentação em tópicos para que possa ser devidamente segmentado.
- Hands-on:
    - Apresente-se
    - Deverá abrir para dúvidas ao final de cada tópico.

## Importância do projeto
 Este projeto possui a importância de abrir espaço para todos da empresa que quiserem compartilhar conhecimentos possam fazê-lo sem nenhum tipo de receio. É um espaço livre de julgamentos em que os aspirantes à palestrantes possam dar seus primeiros passos, e para que os mais experientes possam validar e compartilhar sua vivência.
